Venez découvrir comment écrire du code de bonne qualité en jouant une partie de “Buy a feature</em>” autour des “mauvaises odeurs”.

Si vous avez déjà écrit ou lu du code en ayant la sensation que quelque chose n’allait pas, le terme de “code smell</em>” inventé par Kent Beck</span> vous sera familier.
Les mauvaises odeurs les plus courantes ont été cataloguées pour que tout développeur puisse identifier des problèmes dans son code et connaître les solutions qui vont avec.

Nous proposerons aux participants de choisir des thèmes concrets de CSS</abbr>, HTML</abbr>, Javascript, Python ou PHP</abbr> qui nous serviront à illustrer des “code smell</em>” que les évolutions technologiques ont parfois supprimés afin de vous donner des pistes pour écrire un code plus facile à maintenir.
